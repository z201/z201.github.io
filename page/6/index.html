LinkedList 是是一个有序的集合，是Collection的实现类之一。

概述

  • 双链表 LinkedList 是 Java Collecion Framework成员之一。
  • 双链表 是List和Deque的接口的实现。
  • 在内部，它是使用双链表数据结构实现的。
  • 它支持重复的元素。它没有实现RandomAccess接口。所以我们只能按顺序访问元素。它不支持随机访问元素。
  • 相对与ArrayList它 删除 和添加 速度会快很多。但是遍历就要慢一些。
  • 它以插入顺序存储或维护它的元素。

Collection 这个接口非常重要。List、Set 、Queue的顶层接口都是它，熟悉它对后面的学习会有非常大的帮助。

从上图可以看到Collection接口是多有的重要,抽象了大量的方法供集合类使用

概述

一个集合、即Collection。一般叫它容器，就是将多个相同类型的元素聚集在一起的单元数据。Collection通常用来存储、检索、操作聚集的数据以及聚集数据之间的通讯。

  • Collection集合层次结构中的根接口。一个集合表示一组对象，称为其元素。有些集合允许重复元素，而其他集合不允许。一些是有序的，另一些是无序的。
  • 这样设计的优点也十分显著，减少了学习API的负担。学习了Collection接口在看实现类会轻松很多。

泛型E

​ 该接口封装了不同类型的结合，使得开发人员操作集合的时候不必关心他们具体的实现，核心的集合接口是Java集合框架的基础。证如上图看到的核心集合接口的层次关系。

​ 语法告诉我们这个接口是泛化的，在使用集合的时候需要声明集合的实例、指定集合中对象的类型。指定类型之后编译器能在编译期间验证集合中的元素类型是正确的。从而减少错误。
