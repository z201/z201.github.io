初探Collection

发表于 2018-04-08 | 分类于 Java基础
字数统计: 806 | 阅读时长 ≈ 3

Java集合类大纲

Collection 这个接口非常重要。List、Set 、Queue的顶层接口都是它，熟悉它对后面的学习会有非常大的帮助。

从上图可以看到Collection接口是多有的重要,抽象了大量的方法供集合类使用

概述

一个集合、即Collection。一般叫它容器，就是将多个相同类型的元素聚集在一起的单元数据。Collection通常用来存储、检索、操作聚集的数据以及聚集数据之间的通讯。

  • Collection集合层次结构中的根接口。一个集合表示一组对象，称为其元素。有些集合允许重复元素，而其他集合不允许。一些是有序的，另一些是无序的。
  • 这样设计的优点也十分显著，减少了学习API的负担。学习了Collection接口在看实现类会轻松很多。

泛型E

​ 该接口封装了不同类型的结合，使得开发人员操作集合的时候不必关心他们具体的实现，核心的集合接口是Java集合框架的基础。证如上图看到的核心集合接口的层次关系。

​ 语法告诉我们这个接口是泛化的，在使用集合的时候需要声明集合的实例、指定集合中对象的类型。指定类型之后编译器能在编译期间验证集合中的元素类型是正确的。从而减少错误。

接口类概述

Collection核心都是以接口为基础的，Collection<E>泛型是泛型，当我们要使用的使用的时候需要使用wrapper也就是包装类，比如 Integer 这种封装类型。Colleciton不支持原生数据类型，比如int这种。

  • 同时提供了单独的接口，如果某个集合使用到了不是它这个集合支持的方法将抛出UnsupportedOperationException这个异常。

实现类概述

上图可以看到有Collection有很多实现类下面简单快速的介绍下。

Queue

  • LinkedList
    • 双向列队结构。
  • PriorityQueue
    • 基于堆机构实现，可以用来做优先级列队。

Set

  • HashSet
    • 基于哈希实现，支持快速查找，但不支持有序性操作，例如根据一个范围查找元素的操作。并且失去了元素的插入顺序信息，也就是说使用 Iterator 遍历 HashSet 得到的结果是不确定的。
  • TreeSet
    • 基于红黑树实现，支持有序性操作，但是查找效率不如 HashSet，HashSet 查找时间复杂度为 O(1)，TreeSet 则为 O(logN)；
  • LinkedHashSet
    • 具有 HashSet 的查找效率，且内部使用链表维护元素的插入顺序。

List

  • ArrayList
    • 基于动态数组实现，支持随机访问元素。
  • Vector
    • 和ArrayList类似，但是它是线程安全的。
  • Stack
    • 是Vector的子类，它实现了一个标准的后进先出的栈。
  • LinkedList
    • 基于双向循环列表，只能顺序访问；可以快速的插入和删除元素。而且，LinkedList还可以用作栈、列队和双端列队。
